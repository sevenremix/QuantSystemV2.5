## 一、 核心引擎：三阶判定矩阵 (Logic Confirmation)

Agent 必须遵循严格的布尔串联门控结构。任一判定为 `False` 即触发全链路熔断，信号转为 **Neutral**。

1. 宏观门 (Macro Gate) —— 燃料审计

- **数据源**：FRED $DFII10$ (10年期实际利率) 。
- **处理逻辑**：应用 **3日 EMA (指数移动平均)** 过滤瞬时噪音 。
- **计算公式**：$V_{drop} = \text{Peak}_{20} - \text{EMA}_3(DFII10)$ 。
- **开启标准**：$V_{drop} > 0.1\%$ 时 `Macro Bullish = True` 。

2. 市场门 (Market Gate) —— 情绪审计

- **数据源**：$QQQ$ (20日 Z-Score) 。
- **判定公式**：$QQQ\_Z > -0.5$ 时 `Market Confirmed = True` 。
- **动作**：若判定为 `False`，禁止开立新仓 。

3. 个股门 (Stock Gate) —— 价值审计

- **入场逻辑**：$(Forward\_PE < PE_{threshold}) \land (Z \le buy\_z) \land (PEG \le 1.0)$ 。
- **含金量校验**：$PEG = \frac{Forward\_PE}{g_{expected} \times 100}$ （注：$g_{expected}$ 即为预期 EPS 复合增速，例如 0.8 代表预期业绩增速 80%）。只有 $PEG \le 1.0$ 才通过审计 。

## 二、 退出引擎：动态执行策略 (Dynamic Exit)

系统通过 **动作调节系数 ($C_{action}$)** 实施“部分止盈/止损”，而非单一开关 。

1. 动态比例计算公式

$$C_{action} = \min(1.0, R_{base} \times W_{macro} \times W_{market})$$

- **$R_{base}$ (基础比例)**：字典预设的阶梯减仓百分比（如 $0.5$ 代表计划卖出持仓的 $50\%$）。缺失时调取全局后备 `GLOBAL_DEFAULT_R_BASE = 0.33`。
- **$W_{macro}$ (宏观权重)** ：
    - 若 $V_{drop} \ge 0.17\%$ (势能极强)，令 $W_{macro} = 0.5$ (**惜售模式**) 。
    - 若 $0 < V_{drop} < 0.17\%$ (势能一般)，令 $W_{macro} = 1.0$ 。
    - 若 $V_{drop} \le 0$ (势能衰退)，令 $W_{macro} = 1.5$ (**警力增加**) 。
- **$W_{market}$ (市场权重)** ：
    - 若 $QQQ\_Z > 0$ (强势)，令 $W_{market} = 0.8$ 。
    - 若 $-0.5 \le QQQ\_Z \le 0$ (震荡中轨)，令 $W_{market} = 1.0$ (中性) 。
    - 若 $QQQ\_Z < -0.5$ (转弱)，令 $W_{market} = 1.5$ (**防御增强**) 。

2. 执行优先级

- **逻辑止损 (PE-based)**：优先级**极高**。一旦 $PE > sl\_pe\_max$，直接执行 $100\%$ 清仓（$C_{action} = 1.0$），不受宏观权重调节 。
- **技术止盈/止损 (Z-based)**：优先级**中**。严格受 $C_{action}$ 动态公式调节，实现平滑减仓 。
